## ⚠️ 重要提示

### **第一次使用搜尋時**：

**必須點擊「重建搜尋索引」！**

因為：
- 索引檔案還不存在
- 或者舊索引結構不對

點擊後會：
- 自動刪除舊索引
- 建立新表格（含 `cache_key`）
- 索引所有 10 萬條翻譯
- 耗時約 1~2 秒

### **之後什麼時候需要重建索引？**

✅ **需要重建**：
- 執行翻譯任務後（新增大量翻譯）
- 搜尋結果不正確時
- 快取資料有明顯變動

❌ **不需要重建**：
- 只是查詢現有翻譯
- 小量修改（< 100 條）

## 📝 技術細節

### 修正前的問題

**錯誤訊息**：
```
sqlite3.OperationalError: table cache_fts has no column named cache_key
sqlite3.OperationalError: no such table: cache_basic
```

**原因**：
- 舊索引檔案使用舊表格結構（沒有 `cache_key` 欄位）
- 搜尋引擎初始化後無法動態修改表格結構

### 修正後的行為

**`rebuild_search_index()` 流程**：
```
1. 關閉舊搜尋引擎連線
2. 刪除 search_index.db 檔案
3. 重新初始化搜尋引擎
   → 偵測到沒有檔案
   → 建立新表格（含 cache_key）
4. 索引所有快取資料
5. 完成！
```
