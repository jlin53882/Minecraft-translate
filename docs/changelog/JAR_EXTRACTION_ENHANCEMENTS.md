# JAR 提取器功能強化

**日期：2026-02-14**  
**版本：1.1**

## 新增功能

### 1. 預覽模式（高優先級） ✅

#### 功能描述
- 在實際提取前，先掃描 JAR 檔案內的目標檔案
- **即時顯示進度條**（掃描中 120/460...）
- 顯示將要提取的檔案數量和大小資訊
- **自動生成詳細報告**到輸出資料夾
- 提供確認對話框，避免誤操作

#### 使用方式
1. 在提取器頁面選擇 Mods 資料夾
2. **（選填）設定輸出資料夾**（有設定才會生成報告）
3. 點擊「預覽 Lang」或「預覽 Book」按鈕
4. 系統開始掃描，即時顯示進度
5. 掃描完成後：
   - 彈出預覽對話框
   - 顯示摘要統計（總檔案數、大小、JAR 清單）
   - **如果有設定輸出路徑，自動生成 Markdown 報告**
6. 確認後可直接開始提取

#### 技術實作
- 新增 `jar_processor.preview_extraction_generator()` - generator 版本可即時 yield 進度
- 新增 `jar_processor.generate_preview_report()` - 生成 Markdown 報告
- 新增 `services.preview_jar_extraction_service()` 服務層
- UI 使用**背景執行緒 + 輪詢器**即時更新進度條
- 對話框使用 **`page.open()` / `page.close()`** 配對方式（修復背景執行緒顯示問題）

---

### 2. 提取結果摘要（中優先級）

#### 功能描述
- 提取完成後自動顯示摘要對話框
- 統計資訊：
  - 成功提取的 JAR 數量
  - 跳過的 JAR 數量（檔案已存在）
  - 失敗的 JAR 數量
  - 總共提取的檔案數

#### 使用方式
- 提取任務完成後自動彈出摘要對話框
- 一目了然地檢視提取結果

#### 技術實作
- 在 `ExtractorView` 中新增 `_extraction_stats` 統計
- 從 log 訊息中解析統計資料
- 提取完成時自動顯示摘要

報告檔名：`preview_report_{mode}_{timestamp}.md`  
範例：`preview_report_lang_20260214_123045.md`

**報告內容：**
```markdown

- **總 JAR 數量：** 460

- **有檔案的 JAR：** 360
- **總大小：** 125.50 MB

## 詳細清單
### 1. ae2-1.21.1.jar
- **檔案數量：** 3

  - `assets/ae2/lang/en_us.json`
  - `assets/ae2/lang/zh_cn.json`
  - `assets/ae2/lang/zh_tw.json`
...
```

## Bug 修復記錄

### 問題 1：背景執行緒無法顯示對話框
**症狀：** 預覽完成後對話框不彈出

**原因：** 使用 `page.overlay.append()` + `dialog.open = True` 方式，背景執行緒無法正確更新 UI

**解決：** 改用 Flet 官方推薦的配對 API
- 開啟：`page.open(dialog)`
- 關閉：`page.close(dialog)`

### 問題 2：關閉對話框後灰色遮罩殘留
**症狀：** 點擊「取消」後畫面變灰無法操作

**原因：** 手動操作 `page.overlay` 和 `dialog.open` 導致遮罩層未清理

**解決：** 使用 `page.close(dialog)` 自動管理遮罩層

## 使用建議

### 預覽模式最佳實務
1. **處理大量 JAR 時必用**
   - 超過 20 個 JAR 檔案建議先預覽
   - 確認提取數量符合預期

2. **避免誤操作**
   - 檢查輸出路徑是否正確
   - 確認將要提取的檔案類型

3. **效能考量**
   - 預覽掃描比實際提取快很多
   - 適合用來快速檢查 JAR 內容

### 結果摘要解讀
- **成功**：順利提取新檔案的 JAR
- **跳過**：檔案已存在且內容相同（正常行為）
- **失敗**：讀取錯誤或其他異常

---

## 注意事項

1. **預覽不會修改任何檔案**
   - 純掃描，安全無副作用

2. **統計從 log 解析**
   - 統計數字可能不完全精確（依賴 log 格式）

3. **背景執行緒**
   - 預覽使用背景執行緒避免 UI 卡頓
   - 大量 JAR 檔案時仍可能需要等待數秒
